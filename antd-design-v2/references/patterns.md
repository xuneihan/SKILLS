# 高频业务页面模式

五种 B端中后台最常见的页面模式，每种包含结构说明、信息层级和关键交互规范。

---

## 模式一：搜索列表页（Search + List）

**适用场景：** 用户管理、订单列表、日志查询、商品管理、审批列表

### 页面结构

```
┌─────────────────────────────────────┐
│  页面标题 + 面包屑                    │
├─────────────────────────────────────┤
│  搜索条件区（Card）                   │
│  [关键词输入] [状态下拉] [日期区间]   │
│                      [重置] [搜索]   │
├─────────────────────────────────────┤
│  列表工具栏                           │
│  [新增按钮]        [批量操作] [导出]  │
├─────────────────────────────────────┤
│  Table 数据表格                       │
│  （固定表头，右侧操作列 fixed=right） │
├─────────────────────────────────────┤
│  Pagination 分页                     │
└─────────────────────────────────────┘
```

### 关键规范

- 搜索条件超过 4 个：使用展开/收起折叠，默认只展示常用条件
- 搜索条件少于等于 4 个：平铺一行，不折叠
- "搜索"按钮是主要按钮（Primary），"重置"是默认按钮（Default）
- 列表上方工具栏：左侧放主动操作（新增），右侧放被动工具（导出、刷新、列配置）
- 选中行后工具栏出现批量操作浮层，取消选中后自动隐藏
- 分页：`pageSize` 默认 20，提供 10/20/50/100 切换；显示 `共 N 条`

### 空态设计

| 原因 | 提示文字 | 操作 |
|------|---------|------|
| 从未添加数据 | "暂无数据，立即添加" | 主按钮 → 新增弹窗 |
| 搜索无结果 | "没有找到匹配的结果，试试修改搜索条件" | 文字按钮 → 重置条件 |
| 权限不足 | "暂无查看权限" | 提示联系管理员，无操作按钮 |

## 模式二：表单页（Form）

**适用场景：** 新增记录、编辑记录、配置设置、申请提交

### 页面结构（字段 ≤ 12 个）

```
┌─────────────────────────────────────┐
│  面包屑：列表页 / 新增              │
│  页面标题                           │
├─────────────────────────────────────┤
│  Form 表单区（Card 包裹）            │
│  标签（84px右对齐）  输入组件        │
│  ...                                │
├─────────────────────────────────────┤
│  操作栏（sticky bottom）            │
│  [取消]  [保存草稿]  [提交]         │
└─────────────────────────────────────┘
```

### 字段超过 12 个时：分组布局

```
┌─────────────────────────────────────┐
│  Card：基础信息                     │
│  姓名 | 手机 | 邮箱 | 部门           │
├─────────────────────────────────────┤
│  Card：账号配置                     │
│  角色 | 权限组 | 有效期             │
├─────────────────────────────────────┤
│  Card：扩展信息（可折叠）            │
│  [展开更多 v]                       │
├─────────────────────────────────────┤
│  Sticky 操作栏                      │
└─────────────────────────────────────┘
```

### 关键规范

- 必填项标记 `*` 在标签前；若全部必填，只标注少数非必填
- 字段宽度按语义，不要全部拉满：姓名 160px、手机 180px、邮箱 240px
- 实时校验：失焦时触发（`validateTrigger="onBlur"`），提交时全量校验
- 错误提示在字段下方，文字 12px，`colorError` 红色，不用弹窗
- 多列布局：两列用 `Col span={12}`（列间距 24px），三列用 `Col span={8}`
- 提交后：按钮进 loading → 成功跳回列表页 + `message.success` → 失败保留表单 + `message.error`

## 模式三：详情页（Detail）

**适用场景：** 记录详情、订单详情、审批详情、用户信息查看

### 页面结构

```
┌─────────────────────────────────────┐
│  面包屑 + 页面标题                  │
│  状态 Badge          [编辑] [删除]  │
├─────────────────────────────────────┤
│  Descriptions（基础信息，bordered） │
├─────────────────────────────────────┤
│  Tabs（多维度信息切换）              │
│  [操作记录] [关联数据] [日志]        │
├─────────────────────────────────────┤
│  Tab 内容区（Table 或 Timeline）     │
└─────────────────────────────────────┘
```

### 关键规范

- 页面顶部显示当前记录的核心状态（Badge 状态点 + 文字）
- 编辑入口：跳转编辑页（字段多）或右侧 Drawer 弹出（字段少）
- Descriptions 两列为主，长文本字段 `span={2}` 跨列
- 操作历史用 Timeline 展示，时间倒序（最新在上）
- 关联数据用子 Table，不要嵌套过深

## 模式四：工作台 / 首页（Dashboard）

**适用场景：** 系统首页、个人工作台、运营概览

### 页面结构

```
┌──────────────────────────────────────────┐
│  欢迎语 + 今日日期                       │
├────────┬────────┬────────┬───────────────┤
│ KPI卡  │ KPI卡  │ KPI卡  │ KPI卡（4列）  │
├────────┴────────┴────────┴───────────────┤
│  趋势图（折线/面积，宽 8/12）  快捷入口   │
├──────────────────────────────────────────┤
│  待处理事项 Table    最近动态 Timeline   │
└──────────────────────────────────────────┘
```

### KPI 卡片规范

```
┌──────────────────────────┐
│  图标（20px）  标题       │
│                           │
│  主数值（24px 粗体）      │
│  环比涨跌（colorSuccess/  │
│  colorError + 箭头图标）  │
└──────────────────────────┘
```

- 数值变化：涨用 `colorSuccess #52C41A` + 向上箭头；跌用 `colorError #FF4D4F` + 向下箭头
- 图表库推荐：Ant Design Charts（基于 G2），与 Token 体系深度集成
- 趋势图 Y 轴从 0 开始，避免视觉夸大涨跌幅

### 待办事项规范

- 按紧急程度分组或排序：逾期 > 今天到期 > 未来
- 点击行跳转到对应详情页或打开 Drawer
- 分页或"加载更多"，不在首屏展示超过 10 条

## 模式五：数据看板（Data Dashboard）

**适用场景：** 数据分析报表、运营大盘、监控面板

### 页面结构

```
┌─────────────────────────────────────────┐
│  看板标题   [时间筛选]  [数据源切换]    │
├──────┬──────┬──────┬────────────────────┤
│ KPI  │ KPI  │ KPI  │ KPI                │
├──────┴──────┼───────────────────────────┤
│  折线图     │  饼图 / 环形图             │
│  (宽 7/12)  │  (宽 5/12)               │
├─────────────┴───────────────────────────┤
│  明细表格（可下钻的汇总数据）           │
└─────────────────────────────────────────┘
```

### 关键规范

- 图表颜色用品牌色阶：主色 `#1677FF` 为第一序列色，依次用 `#36CFC9 / #FF7A45 / #FAAD14`
- 图表 tooltip 样式与整体 Token 保持一致，背景 `colorBgElevated`，文字 `colorText`
- 时间筛选固定位置，当前范围在标题旁显示
- 数据加载：图表区展示骨架占位，不要整页 Loading
- 空数据时图表显示"暂无数据"说明，而非空白区域
- 数值格式：大数字用万/亿单位（1.2万、8.5亿），小数保留 1 位；金额保留 2 位
