# 三、基础控件规范

## 3.1 按钮 Button

- **组件/场景**：主按钮、次按钮、文字按钮、禁用按钮
- **适用场景**：所有页面操作、提交、确认、跳转、取消操作
- **官方标准规范**：按钮分为主要按钮、次要按钮、文字按钮、危险按钮四种基础类型，层级区分明确。
- **尺寸/参数**：
  - 通栏按钮高度：**88rpx**
  - 圆角：**8rpx**
  - 按钮文字：**28rpx**
  - 小型按钮高度：**64rpx**
- **交互规则**：点击有反馈，禁用状态无点击效果；弹窗底部按钮默认左取消、右确认。
- **强制要求**：
  - 关键操作必须使用主按钮
  - 取消、返回次要操作使用次按钮 / 文字按钮
- **禁止设计**：
  - 主按钮过多、页面多个高亮主按钮
  - 按钮尺寸过小导致误触
  - 禁用状态不明显
- **最佳实践**：单页面仅保留**一个核心主按钮**，弱化次要操作，减少用户选择成本。

## 3.2 输入框 Input

- **组件/场景**：表单输入、搜索输入、信息填写输入框
- **适用场景**：登录、表单、搜索、资料编辑页面
- **官方标准规范**：输入框需展示清晰边框、占位提示文字、输入状态区分，支持输入校验反馈。
- **尺寸/参数**：
  - 输入框高度：**80rpx**
  - 内边距左右：**20rpx**
  - 文字：**28rpx**
- **交互规则**：聚焦高亮、失焦恢复默认；输入错误需展示文字提示；支持清空按钮。
- **强制要求**：
  - 必填项需标注
  - 占位文案清晰说明输入规则
- **禁止设计**：
  - 输入框无边框难以识别
  - 无错误提示
  - 输入区域过小
- **最佳实践**：搜索框默认圆角 **16rpx**，表单输入框圆角 **8rpx**，区分场景样式。

## 3.3 单选 / 多选 / 开关

- **组件/场景**：单选框、复选框、开关选择控件
- **适用场景**：选项选择、协议勾选、状态开启关闭
- **官方标准规范**：选择控件状态区分清晰，默认状态、选中状态、禁用状态视觉差异化。
- **尺寸/参数**：
  - 单选 / 复选框尺寸：**40rpx**
  - 搭配文字：**28rpx**
- **交互规则**：点击控件 + 文字区域均可触发选择；开关切换有动态动效。
- **强制要求**：协议勾选必须为用户主动勾选，**禁止默认勾选**。
- **禁止设计**：
  - 选择状态模糊
  - 点击区域过小
  - 隐私协议默认勾选 ⚠️（审核红线）
- **最佳实践**：重要协议、授权选项，留白充足，引导用户主动操作。
