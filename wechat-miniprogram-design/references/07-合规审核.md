# 七、合规 & 审核红线（高危必看）⚠️

> 本章是审核驳回率最高的雷区。回答任何与隐私、授权、弹窗、仿系统 UI 相关的问题，**必须先对照本章红线**。

## 7.1 授权与隐私弹窗规范

- **组件/场景**：隐私授权、位置授权、手机号授权弹窗
- **适用场景**：所有需要获取用户权限、个人信息的小程序
- **官方标准规范**：所有权限获取必须用户主动触发，禁止启动强制弹窗、刷屏弹窗。
- **尺寸/参数**：遵循官方弹窗标准样式
- **交互规则**：用户点击对应功能后，方可唤起授权弹窗。
- **强制要求**：
  - 禁止小程序启动自动弹出隐私授权
  - 必须提供关闭、拒绝选项
  - 隐私协议可随时查看
- **禁止设计**：
  - 开机强制弹窗、无法关闭弹窗
  - 诱导授权
  - 默认同意隐私协议
- **最佳实践**：用户触发对应功能后按需授权，弹窗文案通俗易懂，明确告知授权用途。

## 7.2 UI 审核通用禁忌

- **组件/场景**：全小程序 UI 合规审核
- **适用场景**：所有小程序上线审核
- **官方标准规范**：禁止模仿系统通知、系统弹窗、系统状态栏样式；禁止虚假警示 UI；禁止诱导点击。
- **交互规则**：所有弹窗、提示均可手动关闭，无强制阻塞。
- **强制要求**：所有自定义 UI 不可伪装系统官方提示。
- **禁止设计**：
  - 仿微信系统弹窗、仿手机系统通知
  - 虚假红包、虚假消息提示
  - 无法关闭广告弹窗
- **最佳实践**：所有自定义弹窗标注小程序自有样式，区分系统原生 UI。

## 八、禁忌设计 & 反面案例汇总（上线前自查清单）

- **组件/场景**：全场景小程序设计避雷
- **适用场景**：所有小程序设计、自查、上线验收
- **官方标准规范**：所有设计需遵循微信开放平台 UI 规范，保证用户体验、避免误导用户。
- **交互规则**：无强制阻塞用户操作、无诱导操作交互。
- **强制要求**：
  - 不得篡改、遮挡微信官方原生控件
  - 不得仿系统 UI
  - 不得强制用户操作

### 必查禁忌清单（逐项打勾）

| # | 禁忌项 | 风险 |
|---|---|---|
| 1 | 遮挡胶囊按钮、状态栏、底部安全区 | 审核驳回 |
| 2 | 无限弹窗、无法关闭弹窗 | 审核驳回 |
| 3 | 默认勾选隐私、用户协议 | 合规高危 |
| 4 | 页面多主按钮、视觉层级混乱 | 体验差 |
| 5 | 文字过小（< 24rpx）、排版杂乱、间距不统一 | 体验差 |
| 6 | 仿系统通知、红包、消息弹窗诱导点击 | 审核驳回 |
| 7 | Tabbar 数量超出 2–5 个范围 | 审核驳回 |
| 8 | 无状态反馈（加载、空态、错误态缺失） | 体验差 |

- **最佳实践**：设计完成后对照本清单逐项自查，规避审核驳回风险。
