# 五、内容容器规范

## 5.1 卡片 Card

- **组件/场景**：信息卡片、商品卡片、内容展示卡片
- **适用场景**：内容聚合、商品展示、信息展示模块
- **官方标准规范**：卡片为独立内容容器，分割页面模块，自带统一圆角、留白。
- **尺寸/参数**：
  - 卡片圆角：**12rpx**
  - 内外边距：**30rpx**
  - 轻微阴影
- **交互规则**：整卡可点击或局部点击，点击反馈柔和。
- **强制要求**：卡片边界清晰，与页面背景区分明显。
- **禁止设计**：
  - 卡片无留白、紧贴边界
  - 多层卡片嵌套杂乱
- **最佳实践**：所有同质信息统一卡片样式，保证页面整洁统一。

## 5.2 列表、宫格、折叠面板

- **组件/场景**：数据列表、功能宫格、折叠展示面板
- **适用场景**：功能入口、数据列表、长内容折叠展示
- **官方标准规范**：列表统一行高、间距；宫格数量规整；折叠面板层级清晰。
- **尺寸/参数**：
  - 列表单行高度：**80–90rpx**
  - 宫格常用 **2 / 3 / 4 列**布局
- **交互规则**：列表点击整行反馈；折叠面板点击展开 / 收起，状态可逆。
- **强制要求**：列表分割线统一，无错乱排版。
- **禁止设计**：
  - 宫格数量杂乱
  - 列表高度参差不齐
  - 折叠无状态标识
- **最佳实践**：功能入口优先使用宫格，数据展示优先使用列表。
